  1. 2014.年鉴
    日期: 2014-12-31    changming

    2014 年即将结束，每个人都应该做个总结。

    工作

    完成基于 redis 的瑞知社区的开发，完成 redis 的事务的透明处理，修正了客户端 jedis 的连接问题

    提取了平台组件的服务化改造需求，对部分组件进行了部分服务化的改造，引入了 dubbo 框架，解决了多数据源下的分页实现

    实现基于 redis 的搜索

    实现基于 ruby 的爬虫脚本，抓取问答数据

    提高 web 应用的设计能力，能从全局角度思考应用的架构

    从企业开发转型到互联网企业，进一步的技术挑战

    新公司中，重构了面向移动终端和 web 端的消息服务架构

    重构了数据源连接池，引入 druid

    搭建 gitlab，统一代码管理

    参加了一些一场开源会议和三场架构、技术大会

    开发了系统部分模块的移动 web 版本

    开发了系统的部分统计模块

    个人

    个人 blog，增加了后台管理功能，加入图片上传功能，修改部分样式

    写了 17 篇 blog，比 2013 年多了一篇，算进步了

    年中做过一次体检，各指标正常，身体无恙，还可以继续奋斗

    跑步大概总计有 80 公里的样子，这个数字没有同事的一个月的量多

    个人的几个项目没有更多的代码提交，处于暂停状态

    看了场球幕电影

    读书

    《Netty 权威指南》

    《大型网站系统与 java 中间件实践》

    《构建可扩展的 Web 站点》

    《Ruby 语言入门》

    《孤伐重洋》

    写到这里的时候，突然发现一年下来，其实没有做多少事情，个人能力提升不大，没有学习一门新语言，个人项目也没有多大进展，想想时间都用在了刷微信和微博上了，甚至连玩的地方也没去过几个。

    希望 2015 年可以写下满意的年鉴！

    2014，再见！
